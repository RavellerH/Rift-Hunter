# Roadmap

## Phase 1 — Prototype (Week 1–2)

- [ ] Phaser 3 project setup
- [ ] WASD player movement + camera follow
- [ ] Basic melee attack + dodge roll
- [ ] One small monster with patrol AI
- [ ] WebSocket connection (2 players in same room)
- [ ] Basic health bars

---

## Phase 2 — Alpha (Week 3–4)

- [ ] All 3 monsters (Scuttler, Thornback, Skywing)
- [ ] Full combat abilities (Q, E skills)
- [ ] In-match leveling system
- [ ] Power Shard pickups
- [ ] Gear Buff drops
- [ ] Full matchmaking (8–20 players)
- [ ] Extraction zone timer

---

## Phase 3 — Beta (Week 5–6)

- [ ] Full map with zones and aerial layer
- [ ] PvP full damage enabled
- [ ] Leaderboard + end screen
- [ ] Sound effects and basic UI polish
- [ ] Mobile touch controls (optional)
- [ ] Deploy on Contabo VPS (Docker)

---

## Phase 4 — Launch Prep

- [ ] Performance testing (20 players, 60Hz client)
- [ ] Anti-cheat basics (server-side position validation)
- [ ] Basic cosmetic system (weapon skin)
- [ ] Landing page + share link
- [ ] Soft launch / playtest with small group

---

## Open Design Questions

1. **Perspective** — Top-down 2.5D or full isometric?
2. **PvP balance** — Full damage or knockback-only PvP in final release?
3. **Progression carry** — Does anything persist between matches (cosmetics only, or light meta-progression)?
4. **Monster scaling** — Do monsters scale to player count or stay fixed?
5. **Air combat** — Full vertical layer or just an aerial boss phase?
6. **Lore tie-in** — Connect to existing Monster Hunter lore project or build fresh IP?
